# نظام توثيق المياه - (Offline + Sync)

هذا هو مشروع تطبيق ويب PWA لإدارة وتوثيق توزيع المياه بين المزود والزبائن، يعمل بدون إنترنت (100% Offline) مع إمكانية المزامنة لاحقاً عند توفر اتصال.

## ما الذي تم إنجازه؟

1. **تسجيل مبسط للزبون**: سيتم سؤال الزبون لمرة واحدة فقط عن (اسمه الكامل ورقم الواتساب) ولن يطلب منه مرة أخرى.
2. **سير العمل (المزود)**:
   - يكتب المزود اسم الزبون.
   - ينشئ "توصيلة جديدة".
   - يعطي كود التأكيد (أو يقرأه) للزبون.
3. **سير العمل (الزبون)**:
   - يفتح التطبيق الخاص به (يعمل بشكل Offline).
   - يسجل اسمه لمرة واحدة.
   - يدخل كود الاستلام من المزود ليتم التأكيد.
4. **Cloudflare Worker (للنسخ الاحتياطي والمزامنة)**:
   - تم إنشاء ملفات الخادم `worker/wrangler.toml` و `worker/worker.js`.
   - التطبيق يحاول رفع العمليات عند توفر الإنترنت.

## كيف تنشر المشروع على GitHub و Cloudflare؟

### الخطوة الأولى: تنزيل المشروع
1. من هذه الشاشة (في AI Studio)، اضغط على علامة **الإعدادات (Gear icon)** في الزاوية العليا، ثم اختر **Export source code**.
2. سيتم تنزيل المشروع كملف مضغوط `.zip`.
3. قم بفك الضغط عن الملف وسمِّ المجلد باسم `water` أو أي اسم تختاره.

### الخطوة الثانية: نشر الواجهة (Frontend) على GitHub Pages
1. ادخل إلى حسابك في GitHub وأنشئ مستودعاً جديداً (Repository) باسم المشروع، مثلاً: `water-app`.
2. ارفع **جميع ملفات المجلد** إلى هذا المستودع.
3. في المستودع الجديد على GitHub:
   - اذهب إلى **Settings > Pages**.
   - اختر المصدر (Source) ليكون GitHub Actions (سيتطلب إعداد workflow لدعم بيئة Vite، ويمكنك أيضاً البناء محلياً عبر `npm run build` ورفع مجلد `dist` فقط على فرع `gh-pages` لتسهيل الأمور).
   - إذا كنت تستخدم أندرويد فقط: يمكنك التسجيل في موقع `Vercel`، وربط حسابك الـ GitHub، وتحديد المستودع الخاص بك. Vercel سيقوم ببناء ونشر المشروع تلقائياً بضغطة زر واحدة وهو أفضل خيار للاستخدام من الجوال!

### الخطوة الثالثة: تفعيل Cloudflare Worker (الـ Backend) بربط GitHub
لقد قمت بتجهيز ملف `worker/wrangler.toml` وملف أتمتة (GitHub Actions) لنشر الووركر تلقائياً من GitHub إلى Cloudflare. للقيام بذلك من هاتفك:

1. سجل دخولك إلى حسابك في [Cloudflare](https://dash.cloudflare.com).
2. من لوحة التحكم، اذهب إلى قسم **Workers & Pages**، ثم إلى **KV** وأنشئ مساحة جديدة (Create Namespace) باسم `WATER_DB`.
3. انسخ الـ **ID** الخاص بتلك المساحة (سيكون نصاً من أحرف وأرقام).
4. اذهب إلى ملف `worker/wrangler.toml` في مستودعك على GitHub (يمكنك تعديله من متصفح الهاتف)، وضع الـ ID مكان `REPLACE_WITH_YOUR_KV_ID`.
5. لتفعيل الرفع التلقائي، تحتاج إلى ربط GitHub بـ Cloudflare عبر الأسرار (Secrets):
   - في Cloudflare، اذهب إلى My Profile > API Tokens وأنشئ واحدًا لديه صلاحية `Edit Cloudflare Workers`.
   - انسخ الرمز (Token) ورقم حسابك (Account ID من واجهة Workers).
   - في GitHub، اذهب إلى **Settings > Secrets and variables > Actions**.
   - أضف سرين (New repository secret):
     - الأول: `CLOUDFLARE_API_TOKEN` وقيمته هو الـ Token الذي نسخته.
     - الثاني: `CLOUDFLARE_ACCOUNT_ID` وقيمته هو رقم الحساب المشفر.
6. الآن بمجرد حفظ ملف `wrangler.toml`، سيقوم GitHub تلقائياً برفع وتحديث الـ Worker نيابة عنك! (يمكنك متابعة ذلك من تبويب Actions في GitHub). 
7. بعد أن ينتهي، سيظهر لك رابط الووركر الخاص بك (مثال: `https://water-service-api.xxx.workers.dev`).

### الخطوة الرابعة: ربط الـ Worker بالتطبيق (Frontend)
قبل أن تقوم بالتحديث النهائي (Commit) على GitHub، يجب أن تخبر الواجهة عن رابط الـ Worker.
- في مشروعك، أنشئ ملف `.env` وقم بكتابة:
  ```env
  VITE_WORKER_URL="https://رابط-الووركر-الخاص-بك.workers.dev"
  ```
- بعدها قم بالبناء مجدداً وسيتولى التطبيق الرفع التلقائي للعمليات إلى سيرفرك.

> **ملاحظة:** التطبيق سيعمل ١٠٠٪ بشكل داخلي Offline بدون كلاود فلير، ولكن ربطك للكلاود فلير سيعطيك نسخاً احتياطية مركزية تحميك من ضياع البيانات في حال مسح المتصفح.
